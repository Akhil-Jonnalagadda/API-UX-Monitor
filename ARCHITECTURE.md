## Component Responsibilities

### Backend Services

**Synthetic Checker**

- Runs HTTP checks every N seconds
- Measures latency accurately
- Handles timeouts and errors
- Stores results in database
- Emits WebSocket events

**Incident Detector**

- Analyzes check results
- Detects failure patterns
- Creates incidents automatically
- Auto-resolves incidents
- Tracks incident lifecycle

**Alert Evaluator** (Future)

- Evaluates alert rules
- Checks thresholds
- Sends notifications
- Manages alert states

### Frontend Pages

**Dashboard**

- Summary metrics cards
- Latency charts (D3.js)
- Latest checks table
- Period filter (1h, 24h, 7d, 30d)

**Endpoints**

- List all endpoints
- Add new endpoints
- Edit endpoint config
- Delete endpoints
- Enable/disable monitoring

**Incidents**

- Incident timeline
- Filter by status
- Incident replay viewer
- Manual resolution
- Statistics cards

**Alerts**

- List alert rules
- Create rules
- Configure thresholds
- Enable/disable rules

## Technology Choices

### Why These Technologies?

**Backend: Node.js + Express**

- JavaScript/TypeScript familiarity
- Large ecosystem (npm)
- Good for I/O-bound tasks (monitoring)
- Easy to deploy

**Database: PostgreSQL + Prisma**

- Robust for time-series data
- Excellent query performance
- Type-safe ORM (Prisma)
- Easy migrations

**Frontend: React + Vite**

- Fast development experience
- Component reusability
- Large community
- Modern tooling

**Charts: D3.js**

- Flexible and powerful
- Beautiful visualizations
- Full control over rendering
- Industry standard

**UI: Bootstrap**

- Quick professional look
- Responsive out of box
- Well documented
- Familiar to recruiters
